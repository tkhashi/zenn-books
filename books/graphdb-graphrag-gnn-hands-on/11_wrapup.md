## 本書で体験したこと

ここまで、映画レコメンデーションという共通テーマで3つのグラフ技術を体験しました。

```
MovieLens データ（評価・映画メタ）
        ↓
[GraphDB: Neo4j]     → Cypher で関係を検索・推薦
        ↓
[GraphRAG: LlamaIndex] → テキストをグラフ化して Q&A
        ↓
[GNN: PyTorch Geometric] → グラフ構造を学習して推薦
```

## 3技術の比較

| 観点 | GraphDB | GraphRAG | GNN |
|------|---------|---------|-----|
| **入力** | 構造化データ（評価・属性） | テキスト | グラフ構造 |
| **何をするか** | 保存・検索・関係探索 | テキスト→グラフ化→Q&A | 機械学習で予測 |
| **出力** | クエリ結果 | 自然言語の回答 | スコア・推薦リスト |
| **LLM 必要** | 不要 | 必要（エンティティ抽出・回答） | 不要 |
| **学習フェーズ** | なし | なし（インデックス構築のみ） | あり（訓練が必要） |
| **説明性** | 高い（クエリが明示的） | 中程度 | 低い（ブラックボックス） |
| **得意なシナリオ** | 関係の探索・集計 | 文書からの知識抽出 | 大量の行動データからの予測 |

## 使い分けの指針

**GraphDB を使うとき**
- 構造化された関係データ（人・組織・取引など）を管理したい
- クエリで関係を柔軟に探索したい
- 説明可能な推薦ロジックが必要なとき

**GraphRAG を使うとき**
- 大量の文書・ドキュメントから知識を抽出したい
- 「このドキュメントに書いてある A と B の関係は？」という質問に答えたい
- RAG の回答精度を上げたい

**GNN を使うとき**
- ユーザーの行動履歴（クリック・購入・評価）が大量にある
- 明示的なルールを書かずに傾向を学習させたい
- 予測精度を最大化したい（説明性よりも精度優先）

## 組み合わせると強い

実際のシステムでは3技術を組み合わせることもあります。

例：EC サイトのレコメンデーション

```
[GraphDB]    商品・ユーザー・カテゴリの関係を管理
[GraphRAG]   商品説明文から特徴・用途を抽出してQ&A対応
[GNN]        購買グラフを学習して「次に買うもの」を予測
```

## 次のステップ

### GraphDB を深める

- Neo4j の Graph Data Science ライブラリ（ページランク・コミュニティ検出）
- [Neo4j 公式チュートリアル](https://neo4j.com/docs/)

### GraphRAG を深める

- Microsoft GraphRAG ライブラリ（コミュニティ集約・グローバルクエリ）
- LlamaIndex の他のグラフストア（Amazon Neptune、Memgraph）

### GNN を深める

- GAT（Graph Attention Network）: 隣人に重みをつけて集約
- 異種グラフ（HeteroGNN）の本格実装
- PyTorch Geometric の公式チュートリアル
- [Stanford CS224W: Machine Learning with Graphs](https://web.stanford.edu/class/cs224w/)

## 環境の片付け

ハンズオンが終わったら Docker コンテナを停止できます。

```bash
# 停止（データは保持）
docker compose down

# 完全削除（データも消す）
docker compose down -v
```
