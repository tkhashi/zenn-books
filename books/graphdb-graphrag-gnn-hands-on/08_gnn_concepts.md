## ニューラルネットワークの超基礎

ニューラルネットワーク（NN）は、入力データを変換して出力を予測するモデルです。

```
[入力] → [重み付き変換] → [出力]
```

たとえば「映画の評価スコアを予測する」場合：

- 入力: ユーザーの属性（年齢・性別）、映画の属性（ジャンル・公開年）
- 出力: 評価スコア（1〜5）

通常の NN はこれらを **独立した特徴量** として扱います。「このユーザーはこの映画に何点つけるか」を個別に予測します。

## グラフ上での NN ：隣人の情報を集める

GNN（Graph Neural Network）は、通常の NN と違い、**ノードの周辺（隣人ノード）の情報も使って予測** します。

```
通常のNN:
ユーザー196 の属性 → [NN] → 評価予測

GNN:
ユーザー196 の属性
+ ユーザー196 が評価した映画の情報    → [GNN] → 評価予測
+ その映画を評価した他ユーザーの情報
```

「同じ映画を好む人たちの特徴」が自然に学習に含まれます。

## 情報伝播のイメージ

GNN は複数の「レイヤー」を積み重ねます。

```
1レイヤー目: 直接の隣人（1ホップ）の情報を集める
2レイヤー目: 隣人の隣人（2ホップ）の情報を集める
```

映画推薦の場合：

- **1ホップ**: ユーザー196 が評価した映画
- **2ホップ**: その映画を評価した他のユーザー
- **3ホップ**: その他のユーザーが評価した別の映画

レイヤーが深くなるほど、遠くの情報を取り込みます。

## リンク予測とは

本書で解くタスクは **リンク予測（Link Prediction）** です。

グラフ上に存在しないエッジ（リンク）が、将来どれだけ生成されそうかをスコアリングします。

```
(ユーザー196) --[まだ評価していない]--> (映画: Star Wars)
                                              ↑
                          このエッジが生じる確率を予測する
```

つまり「ユーザー196 が Star Wars を評価するとしたら、高い評価をしそうか」を予測し、スコアが高い映画を推薦します。

## 本書のアーキテクチャ概要

```
Neo4j のグラフ（943 ユーザー × 1682 映画）
          ↓
PyTorch Geometric の HeteroData 形式に変換
          ↓
GraphSAGE（SAGEConv）モデルで特徴量を学習
          ↓
ユーザー・映画のペアのスコアを計算
          ↓
スコアが高い映画をユーザーに推薦
```

**HeteroData（異種グラフ）** を使うのは、ノードの種類が「ユーザー」と「映画」の2種類あるためです。同じ種類しかない場合は通常の Data を使います。

## 次章で使うモデル: GraphSAGE

GraphSAGE（`SAGEConv`）は、隣人ノードをサンプリングして集約する手法です。

特徴：
- 大規模グラフでも動く（全隣人を使わず一部をサンプリング）
- ミニバッチ学習ができる
- CPU でも現実的な時間で学習できる

次章でコードを書きながら具体的に理解していきます。
