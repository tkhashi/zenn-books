## この章でやること

- ローカルの画像ファイルをモデルに渡してチャットする
- UI レビュー、OCR、JSON 化を体験する

---

:::message
**コース別モデル名**
この章の例コマンドは `gemma4:e4b` を使用しています。
他のコースの方は以下のモデル名で読み替えてください。

| コース | 使用するモデル |
|---|---|
| Lite（e2b に切り替えた方） | `gemma4:e2b` |
| Lite / Standard | `gemma4:e4b` |
| Standard（qwen3.6 あり）/ Pro | `qwen3.6:27b` |
| Pro（比較用） | `gemma4:26b` または `gemma4:31b` |
:::

---

## 画像を準備する

以下のどれかを用意します。

| 種類 | 試せること |
|---|---|
| Web アプリのスクリーンショット | UIレビュー、コンポーネント分割の提案 |
| ダッシュボード画像 | グラフ読み取り、示唆の抽出 |
| 手書きメモ | OCR、要約 |
| 設計図・構成図 | データフロー説明、Mermaid 化 |
| 帳票・請求書 | テキスト抽出、JSON 化 |

macOS では `Shift + Command + 4` で範囲スクリーンショットを撮れます。

以降の手順では画像ファイルを `~/Desktop/sample.png` として説明します。実際のパスに置き換えてください。

---

## 演習 1: 画像の内容説明

```bash
ollama run gemma4:e4b ~/Desktop/sample.png \
  "この画像に何が写っているか説明してください。重要な文字、ボタン、表、問題点を分けて箇条書きにしてください。"
```

**完了条件**

- [ ] 画像内の主要な要素と文字が説明された

---

## 演習 2: UI レビュー

```bash
ollama run gemma4:e4b ~/Desktop/sample.png \
  "このUIをレビューしてください。画面の目的、ユーザー導線、情報設計、アクセシビリティ、文言、改善案の観点で箇条書きにしてください。"
```

**完了条件**

- [ ] 改善点が 3 つ以上出力された

---

## 演習 3: React コンポーネント設計（Standard / Pro コース推奨）

`qwen3.6:27b` を持っている場合は、コーディング観点の分析も試せます。

```bash
ollama run qwen3.6:27b ~/Desktop/sample.png \
  "このUIをReact + TypeScript + Tailwind CSSで実装する場合のコンポーネント構成、props設計、アクセシビリティ上の注意を出してください。"
```

**完了条件**

- [ ] React コンポーネント構成が出力された

---

## 演習 4: 帳票・表の JSON 化

```bash
ollama run gemma4:e4b ~/Desktop/sample.png \
  "この画像から読み取れるテキスト、数値、表構造を抽出してください。最後にJSON形式で構造化してください。読み取れない箇所は \"unknown\" にしてください。"
```

**完了条件**

- [ ] JSON 形式の構造化データが出力された
