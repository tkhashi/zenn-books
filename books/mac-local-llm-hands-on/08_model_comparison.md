## この章でやること

- 同じプロンプトを複数モデルに投げる
- 速度・品質・コード品質・画像理解を主観で記録する

---

:::message
**複数モデルを持っていない方はスキップして構いません。**
Lite コースで `gemma4:e4b` のみの方は、1モデルの印象をメモするだけでも OK です。
:::

---

## 比較プロンプト

以下を各モデルに投げます。Continue のチャットパネル、または `ollama run` で実行してください。

```text
以下の要件を満たす TODO アプリの設計をしてください。

条件:
- React
- TypeScript
- LocalStorage 保存
- フィルタリング
- 編集
- 削除
- テストしやすい設計

出力:
1. コンポーネント構成
2. 型定義
3. 状態管理方針
4. 実装コード（主要部分）
5. テスト方針
```

---

## 評価の目安

| 観点 | 何を見るか |
|---|---|
| **初回応答の速さ** | 最初の 1 文が出るまでの体感時間 |
| **回答品質** | TypeScript の型・関数責務の妥当性 |
| **コード品質** | そのまま貼って使えるコードに近いか |
| **画像理解** | 画面要素を正しく読めているか |
| **メモリ圧迫感** | Activity Monitor のメモリプレッシャー |

---

## モデルを切り替えるときの手順

大型モデルを使っている場合は、切り替えの前に停止します。

```bash
# 現在起動中のモデルを確認
ollama ps

# 停止
ollama stop gemma4:e4b

# 次のモデルを起動（起動は run で自動）
ollama run qwen3.6:27b
```

確認できたら、次章（トラブルシューティング）に進みます。
