## この章でやること

- Continue のチャットパネルで `src/todo.ts` を分析・改善・テスト生成させる
- UI スクリーンショットから実装案を作る

---

VS Code で `~/local-llm-hands-on/src/todo.ts` を開いた状態で、Continue チャットパネル（`Cmd + L`）から以下を試します。

---

## 演習 1: コード説明

`src/todo.ts` を選択してから、以下をチャットに入力します。

```text
このファイルの責務を説明してください。

観点:
- 型定義
- 各関数の責務
- 副作用
- バグになりそうな箇所
- テストすべき観点
```

**完了条件**

| 確認項目 | 期待する回答 |
|---|---|
| 責務 | Todo 操作のユーティリティであることが説明される |
| 副作用 | `Date.now()` と `new Date()` が指摘される |
| テスト観点 | 空文字、active/done/all フィルタ、存在しない id が出る |

---

## 演習 2: リファクタリング

```text
このコードをリファクタリングしてください。

条件:
- 振る舞いは変えない
- filter の型を string ではなく union type にする
- id 生成と現在時刻をテストしやすくする
- 変更理由を説明する
```

**期待する方向性**

- `type TodoFilter = "all" | "active" | "done"` が作られる
- `addTodo` に `now` や `id` を外から渡せるようになる
- 境界条件が明確になる

---

## 演習 3: テスト生成

```text
このファイルに対する Vitest のテストを書いてください。

含めるケース:
- addTodo で Todo が追加される
- 空文字 title では追加されない
- toggleTodo で done が反転する
- 存在しない id では配列内容が変わらない
- filterTodos で active/done/all を確認する
```

**完了条件**

- [ ] テストファイル（`todo.test.ts` など）が生成された
- [ ] 上記 5 つのケースがすべてカバーされている

---

## 演習 4: UI 画像から実装案を作る（任意）

前章で使った UI スクリーンショットを Continue に貼り付け、以下を入力します。

```text
このUIスクリーンショットをもとに、React + TypeScript + Tailwind CSS で実装してください。

出力:
1. コンポーネント構成
2. props 設計
3. 実装コード
4. アクセシビリティ上の注意
5. レスポンシブ対応
```

:::message
画像を Continue チャットパネルに貼り付けるには、`Cmd + V` または画像ファイルをドラッグ＆ドロップします。
image_input 対応モデルが選択されていることを確認してください。
:::
